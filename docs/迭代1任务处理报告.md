## 七、文档 vs 源码交叉校验报告

> 对比维度：pages.json 路由 / routes.ts / API 接口 / 类型定义 / US 验收标准 / 组件规划

### 阻断项（7 项 — 对接真实 API 前必须修复）

| 编号 | 问题 | 涉及文件 |
|:----:|------|----------|
| B1 | **入口页顺序错误** — `pages.json` 中 `home` 排在第一位（带 `type:home`），`launch` 排在第二。uni-app 以第一个为入口，启动页逻辑被跳过。 | `src/pages.json` |
| B2 | **HTTP 层丢弃 pagination** — `request()` 仅 `resolve(body.data)`，分页接口丢失 `next_cursor` 和 `has_more`。 | `src/services/http.ts` |
| B3 | **错误码匹配断裂** — `request()` reject 时传 human-readable 文本，但 login/register 页 ERROR_MSG 以数字错误码为 key，映射永远不命中。 | `http.ts` + login/register |
| B4 | **分页方式不符** — API 文档要求游标分页（`cursor`），前端用 `slice()` offset 分页。 | `home/index.vue`、`search/index.vue` |
| B5 | **Auth 服务未对接 API** — `mockLogin`/`mockRegister` 绕过 `http.ts`，token 刷新/退出登录/注册 API 均未调用。 | `services/auth.ts`、`stores/auth.ts` |
